How these numbers are calculated
  • Received — sum of approved payment submissions (live data, not cached profiles).
  • Expected — catalog language fee × each student’s current level in the batch.
  • Pending / overdue — same rules as the main Payment Hub student table.
  • LKR collection % — received LKR ÷ expected LKR for that batch (INR/Euro shown separately in columns).
  • Journey day — batch calendar day from journey config when set; otherwise highest student day / level total (e.g. 23/42).
  • Student breakdown — per-student A1–C2 paid, docs, visa, pending, and status.
